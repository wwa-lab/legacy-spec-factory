# Example: Partial Case - Missing Mutating Program Analysis

## Overview

This example shows a data model package that can document structure from DDS
but cannot complete CRUD/lifecycle analysis because a mutating program lacks an
approved program analysis.

## Scenario

- **Domain**: Invoice header and batch posting
- **Files**:
  - INVHDR (PF) - DDS source and DB2 metadata are present
  - INVBYDT (LF) - DDS source is present
- **Programs**:
  - INVINQ - approved program analysis; reads INVHDR
  - INVPOST - listed in inventory and appears to update INVHDR, but no approved
    `program-analysis-<OBJ-ID>.md` exists
- **Inventory**: OBJ-INVOICE-001, OBJ-INVOICE-BY-DATE-001, OBJ-INVINQ-001,
  and OBJ-INVPOST-001 are present and inventory is approved

## What Can Proceed

The structural package can be drafted because DDS and metadata exist:

- `data-dictionary.md` can list INVHDR fields, types, lengths, and DDS line
  evidence.
- `access-paths.md` can list INVBYDT key lines and select/omit criteria.
- `relationship-map.md` can list only relationships confirmed by DDS/DB2, or
  candidates with `needs_sme_review`.

## What Must Be Blocked

The CRUD/lifecycle matrix must not claim update, archive, purge, or posting
behavior for INVPOST until its program analysis is approved.

Correct TBD:

```text
TBD-INVOICE-001: Approved program analysis missing for mutating program INVPOST
- Status: pending_source (blocking for CRUD/lifecycle completion)
- Points to: OBJ-INVPOST-001, OBJ-INVOICE-001
- Evidence observed: inventory relationship says INVPOST updates INVHDR
- Missing evidence: program-analysis-OBJ-INVPOST-001.md with file I/O and error handling
- Required resolver: run legacy-ibmi-program-analyzer for OBJ-INVPOST-001
- Downstream impact: data dictionary may be reviewed; lifecycle and retention
  cannot be approved
```

Incorrect claim:

```text
INVPOST updates INVHDR during batch posting and archives invoices after posting.
```

Why it is wrong:

- It converts an inventory relationship into behavior without source evidence.
- It invents archive behavior.
- It skips the program-analyzer gate required for mutating access.

## Expected Package Status

- `data-model-overview.md`: `status: needs_sme_review`
- `data-dictionary.md`: may be `draft` or `approved` for structural field facts
- `crud-lifecycle-matrix.md`: `blocked` until INVPOST analysis is approved
- `data-model-review-checklist.md`: SME may approve field structure but must
  leave lifecycle approval unchecked

## Recovery Path

1. Run `legacy-ibmi-program-analyzer` on OBJ-INVPOST-001.
2. Verify file I/O statements for INVHDR: READ, WRITE, UPDATE, DELETE, archive,
   purge, or export.
3. Re-run this data-model analyzer against the approved program analysis.
4. Ask SME to decide whether any remaining lifecycle TBDs are blocking.
