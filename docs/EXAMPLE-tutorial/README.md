# EXAMPLE Project — Price Calculation Module

A fully-populated minimal Legacy Spec Factory project. Walk through every
file to see what each stage's artifact looks like in practice.

This is a **synthetic** example — the RPGLE program, fields, and rules
are fabricated. The shape, ID conventions, and traceability are real.

## Scenario

A small AS400 module `PRICE-CALC` calculates the unit price for an order
line based on customer tier, item base price, and an active promotion.
One program (`PRICECALC`), one business flow (calculate price for one
order line), one capability (`CAP-PRICE-CALCULATION`).

## What's NOT in this example

To stay compact, this example omits:

- View 2 / 3 / 4 file bodies (referenced in `module-overview.md` but
  written as one-liner stubs to keep the example small)
- Optional Layer 1 artifacts (DSPF / PRTF / data model)
- Runtime evidence mining (job logs)
- Equivalence pack and forward handoff (stages 9 / 10)
- BRD or modernization decision records

A real production project would have all of those. Use this example for
the **shape and traceability** of each file — not the depth.

## ID conventions used here

- `OBJ-PRICE-CALC-PRICECALC` — one program object
- `OBJ-PRICE-CALC-PRICEFL` — one DB2 file
- `EV-SRC-001`, `EV-DDS-001`, `EV-SME-001` — evidence items
- `FLOW-PRICE-CALCULATE-LINE` — one business flow
- `BR-PRICE-001`, `BR-PRICE-002` — business rules
- `CAP-PRICE-CALCULATION` — one capability
- `TBD-PRICE-001` — open question
