# Partial Case: Async SME Review With Follow-Ups

This case shows an asynchronous review where the SME confirms one behavior,
rejects one inferred rule, and requests more evidence for one TBD. The review
does not produce full advancement; it produces signed decisions plus actionable
follow-up findings.

Use this example when validating async question-pack handling, partial approval,
rule-revision routing, and evidence-gap routing.
